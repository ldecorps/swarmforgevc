## Optimizer recommendations

- **The self-target contract is gitignored local state (recommend to human).** `.swarmforge/contract.yaml` lives under a gitignored path, so a fresh checkout or `git clean` wipes it and the build-start gate silently re-holds `missing` — the pipeline then looks *drained* (empty active) rather than *blocked*, which is easy to misread as "nothing to do." Recommend deciding whether the dogfood self-host should carry a durable/committed contract, or whether the gate should special-case the self-target, so a checkout wipe can't quietly freeze intake. Beyond my routing powers (design/config) — flagging.
- **Recurring "host-only, dark when headless" pattern.** Briefing email (fixed by BL-214/BL-258) and now the cost-health sidecar (BL-272, in flight) are the same class: work that only runs from the VS Code host and goes dark on a headless day. Recommend a one-pass audit for any other host-only emitter before it's discovered surface-by-surface. Two-thirds of the way there already.
- **Cap at 2 is not the throughput constraint.** 18 closes in the window with a single coder serializing the build stage; specifier spec-writing pace and QA turnaround remain the pacing factors. No capacity change recommended.
